# Contributing to KittyFocus

感谢你对 **KittyFocus** 项目的关注！🎉

我们欢迎任何形式的贡献，包括功能开发、Bug 修复、文档完善、测试以及代码优化。请在提交代码前阅读以下贡献指南。

---

## 开发流程

本项目采用 GitHub 协作开发流程，请按照以下步骤参与开发：

1. 查看已有 Issue，或创建新的 Issue 说明需要完成的任务。
2. Fork 本仓库（或创建新的开发分支）。
3. 在对应分支完成开发。
4. 提交 Commit，并使用清晰的提交说明。
5. 提交 Pull Request（PR）。
6. 等待其他成员进行 Code Review。
7. 根据 Review 意见修改代码。
8. 审核通过后，由项目维护者合并至 `main` 分支。

---

## Issue 规范

提交 Issue 时，请尽量说明以下内容：

- 问题描述
- 复现步骤（Bug）
- 功能需求（Feature）
- 预期效果
- 相关截图（如有）

---

## Pull Request 规范

提交 PR 前，请确认：

- 完成了对应 Issue 的开发。
- 代码能够正常运行。
- 不影响已有功能。
- 已删除调试代码和无关文件。
- Commit 信息清晰准确。

PR 描述建议包括：

- 本次修改内容
- 对应 Issue
- 测试情况
- 其他需要说明的问题

---

## Code Review

所有 Pull Request 在合并前都需要经过至少一位团队成员审核，主要检查：

- 功能是否符合需求
- 是否符合项目代码规范
- 是否能够正常运行
- 是否影响已有功能
- 是否存在明显 Bug

审核通过后，由项目维护者完成代码合并。

---

## 代码规范

- 保持代码结构清晰、命名规范。
- 尽量添加必要注释。
- 不提交无关文件或临时代码。
- 保持代码风格一致。

---

## Bug 报告

如果发现 Bug，请通过 GitHub Issue 提交，并尽可能提供：

- 操作环境
- 复现步骤
- 实际结果
- 预期结果
- 截图或日志（如有）

---

## 功能建议

欢迎提出新的功能建议，请先创建 Feature Issue，说明：

- 功能背景
- 功能描述
- 使用场景
- 预期效果

经团队讨论后决定是否纳入后续版本。
